# قائمة فحص التجربة الميدانية — جهاز الصيدلية (P1-M11)

> تُنفَّذ هذه القائمة **على جهاز الصيدلية الفعلي** قبل اعتماد التشغيل التجريبي.
> علِّم كل بند: ✅ ناجح · ❌ فاشل (مع ملاحظة) · ⏭️ مؤجَّل بقرار.
> المرجعية التقنية: `README.md` (التشغيل السريع) و`docs/versions.md` (الإصدارات المثبَّتة).

## ١ — تجهيز الجهاز

- [ ] نظام التشغيل محدَّث، والوقت والمنطقة الزمنية صحيحان (**التاريخ المحلي يحكم ترقيم الفواتير وتقارير اليوم وانتهاء الصلاحية**).
- [ ] Docker مثبَّت ويعمل، وNode 22 LTS + pnpm 10 + Python 3.12 متوفرة (`docs/versions.md`).
- [ ] نسخ `.env.example` إلى `.env` وضبط: `POSTGRES_PASSWORD` قوية، `DATABASE_URL` مطابقة، `PRINTER_HOST` بعنوان الطابعة الشبكية (يُترك فارغاً إن لم توجد طابعة حرارية).
- [ ] الجهاز على UPS (انقطاع الكهرباء وارد — النظام offline-first لكن العتاد يحتاج حماية).

## ٢ — التثبيت والتهيئة الأولى

- [ ] `pnpm install` ثم `docker compose up -d` (PostgreSQL 17 + Redis 7 على localhost فقط).
- [ ] تطبيق الهجرات: `packages/db/scripts/apply-migrations.sh` — **11 هجرة** + بذر RBAC تلقائياً.
- [ ] `python3.12 -m pip install -e "apps/api[dev]"` ثم إنشاء مالك النظام: `python3.12 -m pharmaos_api.cli bootstrap-admin …` — **كلمة مرور قوية تُحفظ لدى المالك فقط**.
- [ ] `bootstrap-branch` باسم الفرع الحقيقي، ثم ضبط إعدادات الفرع من الشاشة: اسم الصيدلية، العنوان، الهاتف، الترخيص، الرقم الضريبي، مقاس الورق **80mm**، رسالة الشكر، تفعيل QR والتوقيع حسب الرغبة.
- [ ] **مفتاح النسخ الاحتياطي**: `python3.12 -m pharmaos_api.cli backup export-key` — يُطبع ويُحفظ **ورقياً في مظروف مغلق خارج الجهاز**. بدونه لا يمكن الاستعادة بعد فقدان الجهاز.
- [ ] بذر الكتالوج: `catalog-seed --file <csv>` (مجموعة CC0 ~25 ألف صنف) أو استيراد ملف الصيدلية.
- [ ] إنشاء مستخدمي التشغيل الفعليين (صيدلاني/كاشير/مدير فرع) من شاشة المستخدمين — **لا يعمل أحد بحساب المالك يومياً**.

## ٣ — العتاد: الماسح

- [ ] مسح **EAN-13** محلي (622…): يظهر الصنف في POS خلال أقل من ثانية.
- [ ] مسح **GS1 DataMatrix** ثنائي الأبعاد من عبوة حقيقية (تتبع EDA): يُحل الدواء عبر GTIN، وفي شاشة الاستلام تُملأ الصلاحية ورقم التشغيلة تلقائياً.
- [ ] الماسح بوضع HID "لوحة مفاتيح" يرسل Enter بعد الكود (وإلا يُضبط من دليل الماسح).
- [ ] كود غير معروف → رسالة واضحة واقتراح البحث بالاسم (F3) — لا صمت ولا انهيار.

## ٤ — العتاد: الطابعة والدرج (M9)

- [ ] الطابعة الحرارية على الشبكة المحلية بعنوان ثابت (DHCP reservation) مطابق لـ `PRINTER_HOST`، منفذ 9100.
- [ ] بيع تجريبي نقدي → **يُطبع الإيصال تلقائياً ويُفتح الدرج**.
- [ ] **العربية على الورق**: أسماء الأصناف والرسائل تظهر سليمة. (دعم العربية يختلف بين الطرازات — إن ظهرت رموز مشوهة يُسجَّل الطراز ويُقيَّد بديل bitmap ضمن مصفوفة العتاد المعتمدة قبل الاعتماد.)
- [ ] الإيصال يحمل: الترويسة من الإعدادات، رقم الفاتورة، الأصناف مجمَّعة، الإجماليات، طريقة الدفع، **المدفوع والباقي** للنقدي، QR (إن فُعِّل)، سطر التوقيع (إن فُعِّل)، القطع الجزئي يعمل.
- [ ] بيع **بطاقة** → يُطبع الإيصال **دون فتح الدرج**؛ إعادة الطباعة (P) لا تفتح الدرج.
- [ ] فصل الطابعة ثم البيع → رسالة فشل واضحة مع بقاء الفاتورة سليمة + إمكانية إعادة الطباعة بعد إعادة التوصيل، وطباعة المتصفح (B) تعمل كبديل.

## ٥ — الدفق الوظيفي الكامل (سيناريو وردية)

- [ ] فتح جلسة كاشير برصيد بداية فعلي (عدّ حقيقي).
- [ ] استلام دفعة بمسح GS1 مع مورّد → تظهر في المخزون بالكمية الصحيحة.
- [ ] بيع بتبديل وحدة (شريط ← قرص/علبة) → الأسعار والخصم من المخزون صحيحة (FEFO: الأقرب انتهاءً أولاً).
- [ ] بيع نقدي بمبلغ مستلَم → الباقي صحيح على الشاشة والإيصال، وملخص الجلسة يتحرك.
- [ ] بيع بلا جلسة (حساب صيدلاني) → يظهر التلميح في POS، ويُحتسب "خارج الجلسات" في تقرير اليوم.
- [ ] تسوية مخزون بسبب، وحجْر دفعة → تختفي من المتاح للبيع فوراً.
- [ ] إغلاق الجلسة بجرد فعلي → الفرق (عجز/زيادة/مطابق) صحيح، وتقرير اليوم (Z) يطابق الواقع.
- [ ] سرعة POS مقبولة عملياً: مسح ← سطر في السلة دون تأخير ملحوظ.

## ٦ — الصمود (أهم اختبارات الاعتماد)

- [ ] **فصل الإنترنت كلياً** وتنفيذ وردية مصغّرة (استلام + بيع + طباعة + إغلاق جلسة) — كل شيء يعمل محلياً.
- [ ] **إعادة تشغيل PostgreSQL** (`docker compose restart db`) أثناء اليوم → عند عودة الخدمة لا فواتير ناقصة، وفحص الإقلاع يتحقق من ثبات الكاش ويصلحه إن لزم (سيناريو M12).
- [ ] **قطع الكهرباء عن الجهاز أثناء بيع** (اختبار متعمد) → بعد الإقلاع: لا فاتورة جزئية، والمخزون سليم (`inventory drift-check` يرجع نظيفاً، rc=0).
- [ ] `python3.12 -m pharmaos_api.cli inventory expiry-sweep` يعمل — ويُجدول يومياً (cron/Task Scheduler) مع `backup create`.

## ٧ — النسخ الاحتياطي والاستعادة

- [ ] `backup create` ينتج ملفاً مشفَّراً في `BACKUP_PATH`.
- [ ] **تمرين استعادة فعلي** على قاعدة خربة: `backup restore-drill --file … --drill-database-url …` → الأعداد مطابقة.
- [ ] جدولة النسخ اليومي، وتحديد وجهة النسخة السحابية الأحادية (حين يُفعَّل D5).

## ٨ — الأمان (تحقق نهائي على الجهاز)

- [ ] الـ API لا يستمع إلا على `127.0.0.1:8000` (يُختبر من جهاز آخر على الشبكة: الاتصال مرفوض).
- [ ] `/docs` و`/redoc` معطَّلان (404).
- [ ] قفل الحساب بعد 5 محاولات دخول فاشلة يعمل، والرسالة واضحة.
- [ ] صلاحيات الأدوار على الشاشات صحيحة: الكاشير لا يرى المستخدمين/الإعدادات، ولا يستطيع إغلاق جلسته (المدير يغلق)، وdata_entry يضيف كتالوج ولا يعدّل.
- [ ] لا مفاتيح ولا كلمات مرور في `.env` غير المسموح به (المفاتيح الحرجة في مخزن النظام)، و`.pharmaos-devkeys/` غير موجود على جهاز الإنتاج.
- [ ] متغير `PHARMAOS_ENV=production` مضبوط (يقفل تجاوز مضيف الطابعة ويمنع مخزن مفاتيح التطوير).

## ٩ — التسليم

- [ ] تدريب سريع (١٥ دقيقة): مسح، تبديل وحدة، F-keys، دفع وباقٍ، فتح/إغلاق الجلسة، ماذا تفعل عند فشل الطباعة.
- [ ] ورقة اختصارات POS مطبوعة بجوار الجهاز: `F2 مسح · F3 بحث · ↑↓ تنقّل · +/− كمية · F6 وحدة · Delete حذف · F4 دفع · P إعادة طباعة · B طباعة متصفح`.
- [ ] مظروف مفتاح النسخ الاحتياطي مسلَّم للمالك وموقَّع عليه.
